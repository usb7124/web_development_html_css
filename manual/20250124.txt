  <form> 태그 사용
    : 웹 페이지를 사용하다 보면 아이디와 패스워드를 입력한다거나 체크하고 고르는 등의
    사용자로부터 입력받아야 하는 정보들이 있습니다.
    이때 사용하는 것이 <form>태그는

    <form> 태그의 속성
      예를 들어 이름과 나이를 입력하는 양식이 있다면, 우리는 키보드로 해당 정보를 입역합니다.
      이름 폼에는 -> 텍스트를 입력
      나이 폼에는 -> 숫자를 입력하거나 / 마우스 휠 돌려서 나이를 입력 / 화살표로 입력

    사용자로부터 입력을 받기 위한 양식을 작성하는 태그 전체를 <from>이라고 하고,
    그 속성으로 이름, 나이 입력 요소, 제출, 초기화 등 정보를 제출하기 위한 태그들을 포함합니다



    n1formtag.HTML
    
    form 내부의 속성
      1. action="" : 우리가 이름 / 나이를 입력했을 떄 연결되는 결과 페이지 
      2. method="" : 가서 보여주도록 method를 고정 POST / GET / PUT / DELETE의
        HTTP method를 의미함.
      3. autocomplete="" : 자동완성 속성 default로 "on"이가 때문에 꺼주고 싶으면

    <input> 태그 : 알던 종류가 너무 다양해서 n1에서 작성한 부분만 다룰 예정
      1. name="" : HTML 상으로 특별한 의미를 갖는 것은 아니고, 서버에서 각 항목을 받을때
      무었에 대한 값인지 구분할 수 있도록 정보를 보내는 용도.
    <label> 태그 : 이상의 <input> 택그와 유기적으로 연결되어있음. <input> 태그에 부여한 id
    속성 값을 <label> 태그의 for 속성에서 불러오기 때문에.
    input-name -> label-for

    <button> 태그 -> 특히 emmet 이용해서 목록을 확인했을 때 제출인지 초기화인지 확인
    가능합니다.

      2) radio : 라디오 버튼이락고 불리는 하나만 선택 가능한 원형 버튼을 클릭해서 선택

  기타 <input>
    사용자가 어떤 파일을 첨부해서 업로드하거나 혹은 내용을 숨기는 것. 이메일 양식을 작성해서 제출하는 것 etc
      -> n6etcinputtag.html

    type :
      1) file : 파일을 첨부할 때 사용
         accept : 원하는 유형의 파일만 업로드할 수 있도록 제어.
         multiple : 파일 여러 개를 한 번에 올릴 수 있도록 하는 설정.

      2) hidden : 무언가 숨겨져 있는 상태입니다.
          사용자로부터 받은 개인 정보나 사용자를 알아서는 안되는 등급의 정보 때문입니다.
          javascript와 함쳐져서 작성하는 편

      3) email : 이메일 형식으로 작성하지 않으면 경고창이 뜹니다.
        다만, 아무 텍스트나 넣고 @만 넣어줘도 정상 제출이 되는 등 좀 허술합니다.
        추후에 자바스크립트를 통해서 정규식을 적용하여 사용하는 편입니다.

    <input> 태그의 공통 속성
      ->

      1) value : placeholder와 달리 어떤 default값을 미리 생성하고 싶을 때 사용.

      2) autofocus : 어떤 페이지가 새로고침됐을 때 클릭하지 않아도 자동으로 어떤 요소가 선택된
      상태를 유지하도록 하는 속성

      3) readonly : 읽기만 가능하고 입력값은 결과로 전송됨(disabled과 차이점입니다.)

      4) disabled : 사용자가 값을 입력할 수 없고 결과가 전송되지도 않습니다.

      5) required : 반드시 인풋창을 채워야 하는 속성 -> 채워진 속성은 전송됩니다.
