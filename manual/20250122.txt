태그의 사용 방식 :
  <태그이름> 내용 </태그 이름>
  여는 태그의      닫는 태그

내용은 여러 줄에 걸쳐서 나올 수가 있는데, 해당 경우
<태그이름>
  내용1
  내용2
  내용3
</태그이름>
의 형태로 작성할 수도 있습니다. -> 한줄 혹은 여러줄에 이렇게 작성하는
방식을 : '문법'이라는 표현을 씁니다.

테스트 관련 태그
  1. b 태그 : bold라는 의미로 태그로 감싸진 부분을 볼드체로 표기
  2. strong 태그 : 브라우저 상에서의 효과는 b 태그와 동일.
    차이점으로, 컴퓨터가 소스 코드를 해석할 때 중요하다고 인지하도록 하는
    태그로서, 구글 및 검색 사이트에서 <strong> 태그를 보고 검색어 분류를
    할 때 이용됩니다.
  3. i 태그 : italic이라는 의미로 태그로 감싸진 부분을 이탤릭체로 표기
  4. em 태그 : 브라우저 상에서의 효과는 italic이라는 태그와 동일.
    차이점으로, 컴퓨터가 소스 코드를 해석할 때 중요하다고 인지하도록 하는
    태그로서, 구글 및 검색 사이트에서 <em> 태그를 보고 검색어 분류를
    할 때 이용됩니다.

  5. 첨자 태그 sup vs. sub  -> 각각 위첨자 / 아래첨자 supsubtag.html 참조

  6. 밑줄 태그 / 취소선 태그 
    u 태그 : underline의 줄임말로 밑줄 표시
    s 태그 : strike의 줄임말로 취소선 표시
    다만 예전에는 u 태그가 밑줄 자체 표시로 html상에서 많이 쓰였습니다.
    하지만 꾸미기를 css가 전담하게 되면서 특정 테스트가 맞춤법이 틀렸다는
    의미로 철자 오류 표시 용도 등으로 좀 변하는 중입니다.
